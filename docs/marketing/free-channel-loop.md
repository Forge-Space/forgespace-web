# Forge Space Weekly Free-Channel Loop

## Cadence

Run once per week, tied to one concrete product proof from the previous 7 days.

## Output pack

1. One short post for a rented channel (LinkedIn first, then X/Reddit/Hacker News when proof is strong)
2. One owned-channel update (GitHub discussion/release note or site update)
3. One CTA alignment check on `forgespace.co` hero and footer
4. One message-spine check (ads + landing + posts use the same promise)

## Weekly execution

1. Pick one proof artifact:
   - New feature shipped
   - Real usage result
   - Before/after with governance or speed gains
2. Publish the short post (LinkedIn first):
   - Problem first
   - Proof second
   - One link to `forgespace.co` with UTM params
   - Use weekly source tags: `utm_source=linkedin` then rotate only if needed
3. Publish GitHub ecosystem update:
   - Repos touched
   - What changed for users
   - Next week roadmap bullet
4. Validate CTA consistency:
   - Hero CTA copy still maps to paid-intent keyword clusters
   - Footer still routes to Siza, GitHub, and Contact Sales
5. Capture outcomes:
   - CTA events in GA4
   - Top referral source for the week
   - Which message spine variant produced better CTA quality

## Message spine template

Use the same 3-line narrative in ad copy, hero copy, and community posts:

1. Problem: AI codegen is fast but often hard to govern for small teams.
2. Promise: Forge Space adds practical governance and visibility without heavy platform overhead.
3. Action: Start at `forgespace.co` and choose Siza, GitHub, or Contact Sales.
